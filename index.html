Schedule Format Guide

Share this with providers so they know exactly how to submit their hours. Consistent formatting means fewer errors.

GOOD Clear dates and times
Mon 3/17: 9:00 AM – 5:00 PM
Tue 3/18: 10:00 AM – 3:00 PM
Wed 3/19: OFF
Thu 3/20: 9:00 AM – 6:00 PM
Fri 3/21: 9:00 AM – 1:00 PM
AVOID Vague or incomplete
I'll be available most of next week.
Probably mornings except Wednesday.
Same as last time I think?

​

What we need in every submission

📅 Specific Dates

Day of week + date (e.g., Mon 3/17). No "next week" or "same as usual."

⏰ Start & End Times

Include AM/PM. Use "OFF" for days not working. No "mornings" or "afternoons."

📝 Notes & Exceptions

PTO, half-days, or schedule changes — call them out explicitly.

Email Scanning & Draft Generation

Establish the foundation: use Gemini to scan Gmail for schedule submissions and automatically identify missing providers. Generate draft reminder and escalation emails ready for review.

  1. Open the scheduling inbox Go to Gmail and navigate to Ashley@fountain.net (or the designated scheduling inbox).
  2. Copy the audit prompt Go to the Tools section on this site and copy the Gemini Audit Prompt.
  3. Run the audit in Gemini Open Gemini, paste the prompt, and provide your inbox contents or screenshots. Gemini will identify which providers have not submitted.
  4. Review the missing providers list Cross-check the output against the active provider roster. Note any providers on PTO or leave.
  5. Draft reminder emails Use the email templates to create reminder messages. Copy the appropriate template (friendly or escalation) based on the deadline.
  6. Review and send Personalize each draft if needed, then send to the missing providers. Log who was contacted and when.

Data Collection & Tracking

Build the tracking infrastructure: create an automated spreadsheet that logs submissions, tracks compliance history, and provides real-time visibility into provider status across cycles.

  1. Create a Google Sheet Set up a new spreadsheet with columns: Provider Name, Email, Cycle Dates, Submission Status, Date Received, and Notes.
  2. Import the provider roster Copy the list of active providers into the sheet. Include their email addresses for automated matching.
  3. Set up Google Apps Script Open Extensions > Apps Script. Paste the automation code that scans Gmail for schedule emails and logs them to the sheet.
  4. Configure email parsing rules Define how to extract provider names from email subjects/senders. Set up pattern matching for "Schedule" or provider name variations.
  5. Add conditional formatting Color-code rows: green for submitted, yellow for pending, red for overdue. Add deadline formulas to auto-calculate status.
  6. Test with sample emails Send test schedule emails and verify they populate correctly. Adjust parsing rules as needed.
  7. Enable daily sync Set up a time-based trigger to run the script automatically each morning, keeping the sheet current.

🏆 Definition of Done

  • Sheet auto-updates within 1 hour of email receipt
  • Zero manual data entry required
  • Historical compliance data available for all providers
  • Status visible at a glance via color-coding
Focus Spreadsheet automation, data parsing, compliance tracking

Deliverables Auto-populated tracker, submission history, reporting dashboard

Full Automation & Integration

Connect all the pieces: automated email sending on schedule, calendar blocking from parsed submissions, OnceHub availability sync, and end-to-end workflow orchestration with minimal manual intervention.

  1. Set up scheduled reminder triggers Configure Apps Script to automatically send reminder emails on Day 3 (friendly) and Day 5 (escalation) of each cycle based on missing submissions.
  2. Build email queue with approval Create a "Pending Emails" tab in the sheet. Automated drafts land here for quick review before batch-sending.
  3. Parse schedules into calendar format Extract dates and times from submitted schedules. Convert to structured data (day, start time, end time) for calendar integration.
  4. Connect to Google Calendar Use Calendar API to create "busy" blocks on provider calendars based on their submitted availability.
  5. Integrate with OnceHub Use OnceHub's API or browser automation to sync provider availability. Block booking pages during scheduled patient hours.
  6. Create escalation rules Define automatic escalation paths: after Day 5, CC supervisor; after Day 7, flag for manual intervention.
  7. Test end-to-end with pilot group Run the full workflow with 3-5 providers for one cycle. Document issues and refine automation rules.
  8. Roll out and monitor Deploy to all providers. Set up a dashboard to track automation success rate, response times, and exceptions.

  • Calendar events created automatically from submissions
  • OnceHub availability synced within same day
  • Admin time reduced from hours to minutes per cycle

FAQ & Edge Cases

Common issues and questions about the schedule collection process.

Common Issues

COMMON

Provider sent schedule to the wrong email

The audit only scans emails sent to Ashley@fountain.net. Ask the provider to forward it to the correct address with "Schedule" in the subject line, then manually mark the sheet as received.

COMMON

Schedule has vague or conflicting times

If a provider says "same as last time" or lists overlapping hours, reply asking for clarification with specific dates. Share the format guide. Mark as received once the corrected version comes in.

SYSTEM

Audit didn't detect a submitted schedule

The audit searches for "Schedule" in the subject line. If a provider used a different subject or sent from a personal email, it won't match. Manually update the tracker.

SYSTEM

Reminder sent to provider who already submitted

This happens if the tracker wasn't updated before the reminder task ran. Apologize briefly and mark the sheet as received so escalation doesn't trigger.

Edge Cases

EDGE CASE

Provider is on PTO for the entire period

Mark them as "Submitted" with a note like "PTO — full period." Block the full period on their calendar and set OnceHub availability to zero. No reminder needed.

EDGE CASE

Provider submits mid-cycle schedule change

Update the calendar events, adjust OnceHub availability, and add a note in the tracker (e.g., "Updated 3/12 — moved Thu to PM only"). No need to re-run the audit.

EDGE CASE

Schedule deadline falls on a weekend

Deadlines that fall on weekends are automatically moved to the following Monday. Reminder timing adjusts accordingly to maintain the 2-day buffer.

Frequently Asked Questions

What's the follow-up cadence?
First reminder goes out 2 days before the due date. Escalation (with Lindsay CC'd) goes out on the due date. This gives providers a buffer while ensuring timely collection.
Can multiple team members run the audit?
Currently scoped to Ashley@fountain.net. For shared responsibility, use the Google Sheet as the source of truth so anyone can see status. The escalation CC to Lindsay provides a second set of eyes.
Where do I send providers for submission instructions?

System Administration

How to manage the provider roster and keep the system up to date.

➕
Adding a new provider
When a new provider joins, add them to two places:
Add their name and email to the Master Roster tab of the Google Sheet
Add them to the providers array in app.js (the audit prompt and tables update automatically)
Ensure they're on the Fountain Provider Schedules calendar
➖
Removing a departing provider
When a provider leaves, remove them cleanly:
Remove from the Master Roster tab (or mark as inactive with a note)
Remove from the providers array in app.js
Archive their historical rows — do not delete, in case records are needed
